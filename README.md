# Credit Card Validator

## Описание

Credit Card Validator - консольное приложение на Go для проверки номеров банковских карт. Программа принимает номер карты, очищает его от пробелов и дефисов, проверяет формат, валидирует номер по алгоритму Луна и пытается определить банк по префиксу из файла `banks.txt`.

## Возможности

- Проверка длины номера карты: от 13 до 19 цифр.
- Проверка, что номер содержит только цифры.
- Поддержка ввода с пробелами и дефисами, например `4000-0000-0000-0002`.
- Проверка контрольной суммы по алгоритму Луна.
- Определение банка по префиксу из файла `banks.txt`.
- Интерактивный режим работы до нажатия `Enter` без ввода номера.

## Установка и запуск

1. Убедитесь, что установлен Go.
2. Откройте каталог проекта:

```bash
cd credit-card-validator
```

3. Проверьте, что рядом с `main.go` есть файл `banks.txt` в формате:

```text
Bank Name,Prefix
```

Пример:

```text
Lunar Bank,4000
Mars Credit Union,5000
```

4. Запустите приложение:

```bash
go run .
```

Или соберите исполняемый файл:

```bash
go build -o credit-card-validator
```

## Примеры использования

Запуск:

```bash
go run .
```

Пример успешной проверки:

```text
Загружено банков:  5
Введите номер карты (или Enter для выхода):4000000000000002
Банк:  Lunar Bank
```

Пример ввода с разделителями:

```text
Введите номер карты (или Enter для выхода):4000-0000-0000-0002
Банк:  Lunar Bank
```

Пример невалидного номера:

```text
Введите номер карты (или Enter для выхода):4000000000000000
Ошибка проверки Луна
```

Выход из программы:

```text
Введите номер карты (или Enter для выхода):
Спасибо, до скорых встреч
```

## Структура проекта

- `main.go` - основной код приложения: загрузка данных банков, чтение пользовательского ввода, валидация номера, проверка по алгоритму Луна и определение банка.
- `banks.txt` - список банков и их префиксов в формате `Название банка,Префикс`.
- `go.mod` - описание Go-модуля проекта.
- `.vscode/launch.json` - настройки запуска и отладки в Visual Studio Code.
